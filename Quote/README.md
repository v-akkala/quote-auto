# SKC Service Quote Generator

Desktop tool for filling the SKC service quote Excel template from user input.

## Run

For a brand-new Windows computer, download this repository as a ZIP, unzip it, then double-click:

```bat
setup_portable_quote_generator.bat
```

The setup file will:

- download and install Python for the current Windows user if Python is not already installed
- create a local `.venv` folder inside this project
- install the required packages from `requirements.txt`
- verify Tkinter, OpenPyXL, and Pillow are available

After setup finishes, double-click:

```bat
run_quote_generator.bat
```

For normal daily use after setup, only run:

```bat
run_quote_generator.bat
```

If the local environment is missing or broken, the run file will call setup to repair it.

## Workflow

1. Fill in the quote suffix after the fixed `SKC2627` prefix, dates, staff names, payment options, payment terms, and customer details.
2. Add as many products as needed. Select a product from the list to edit, duplicate, or remove it.
3. Open the `Tests` tab.
4. Add one or more tests. Test types and standards are editable autocomplete dropdowns; Tab cycles prefix matches and Enter accepts one. If some tests apply only to certain products, turn on product assignment for those tests and select the applicable product(s).
5. Click `Generate Excel Quote`.
6. The generated workbook is saved in `outputs` with the quote number as the file name.

Use `Load Existing Quote` to open a quote previously generated by this app and continue editing from its saved details. The loader is intended for generated SKC quote workbooks, not blank templates or unrelated Excel files.

The blank Excel templates live in `templates/Template Quote.xlsx` and `templates/Template Quote with Discount.xlsx`. Generated quote numbers start at `SKC26270001` and increment after each successful generation unless you type a custom suffix. The generated `outputs` and local `.venv` are ignored by git so the repo is ready to share.

The test type/standard catalog is bundled at `data/test_standards.json`. The autocomplete uses built-in Tkinter controls, so no extra package is needed for it.
